# Golf Tournament App MVP

## Product goal

Build a private tournament app for recurring friend golf events that handles:

* login and user accounts
* tournament creation
* invites by email or join code
* RSVP tracking
* course voting
* date voting
* organizer roles
* group chat inside each tournament
* GTA golf course map

## Product framing

This is not a broad golf app. It is a private event and tournament coordination app with golf-specific workflows.

---

## Core user roles

### 1. Player

* create account
* join a tournament by invite or code
* RSVP yes / no / maybe
* vote in polls
* chat in tournament group
* view course options on map
* see event details and updates

### 2. Organizer

* create tournaments
* invite players
* generate join code
* add co-organizers
* create course and date polls
* send updates
* manage RSVPs
* moderate chat

### 3. Admin

* system-level control
* manage users and edge cases

## MVP features

### Authentication

* email/password signup
* Google sign-in optional later
* password reset
* email verification

### Tournament creation

* title
* format
* description
* tentative season label
* organizer list
* max players optional
* join code
* invite by email

### Invitations and onboarding

* organizer adds emails
* system sends invite email with sign-up link
* user creates account and lands inside tournament
* optional join by code for existing users

### RSVP

* yes / no / maybe
* optional note
* organizer view of status counts
* reminder messaging later

### Polls

* date poll with multiple date options
* course poll with multiple course options
* one vote per player per poll unless organizer allows multiple
* results visible to organizer and optionally to players

### Golf course map

* map of Greater Toronto Area
* markers for golf courses
* organizer can search and add course options to a poll
* course detail card with name, address, website, phone, and estimated drive time later

### Tournament chat

* one group chat per tournament
* real-time messages
* organizer pinning of announcements later
* basic moderation controls

### Organizer controls

* multiple organizers per tournament
* add or remove co-organizers
* send tournament-wide updates
* lock poll voting or close poll

## Integration notes

### Email

Use email for:

* invites
* account verification
* RSVP reminders
* organizer updates

### WhatsApp

Do not make WhatsApp a core dependency for v1 chat.
Use it later for outbound notifications or reminder messages only.

Reason:
WhatsApp production setup and template flows add friction compared with email and in-app chat.

### In-app chat

Launch with in-app chat first.
It gives you the group conversation feature you want without relying on external messaging approvals.

## GTA golf course map approach

### Option A: manual seed database

* preload known GTA courses
* fastest for MVP
* more control over data quality

### Option B: map/search API-assisted

* use Google Places or Mapbox search to find golf courses dynamically
* better long term
* more API cost and implementation detail

### Recommended approach

For MVP:

* start with seeded GTA golf course data in your own database
* show markers on the map
* allow organizer to search and select them
* add API-assisted search later

## Product decisions to protect scope

### Keep in v1

* private tournaments
* invitation flow
* simple polls
* simple chat
* organizer permissions

### Exclude from v1

* live scorekeeping
* payments
* tee sheet generation
* advanced handicap logic
* public tournaments
* native mobile app
