You are scaffolding a production-ready FastAPI + SQLite + React (Vite + Tailwind) app called Golf Day OS. Build only what’s specified. Prioritize clarity, small files, and testability.

0) Goals (MVP)

Private groups with minimal auth (email+password, sessions).

Course directory (local DB) with search + CSV import.

Event creation → course/date polls → apply result → open RSVP with cap + waitlist.

Manual pairings + tee sheet PDF export and CSV export.

Email notifications via console (stub function) for now.

No payments, no scoring.

2.3 Security

Session cookies (HTTP-only) using signed cookie (itsdangerous).

Password hashing with passlib bcrypt.

Dependency get_current_user() based on session cookie.

CSRF token via header X-CSRF-Token on mutating requests (simple random token stored in session).

2.5 Business Rules (implement)

Poll voting: 1 vote per user per poll; allow live tally return.

Apply result tie-break: if top options tie, require owner to choose (provide endpoint param).

RSVP:

If joined count < capacity → status=joined.

Else create waitlist entry with position_int = max + 1.

On withdraw: promote waitlist #1 → create claimed_expires_at = now + 24h.

/claim checks expiry windows and promotes next if expired.
